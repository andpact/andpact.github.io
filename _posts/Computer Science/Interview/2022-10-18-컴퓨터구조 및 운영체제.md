개인적인 의견이 포함되어 있고, 잘못된 정보가 있을 수 있습니다.

## **컴퓨터**
- 이진수(비트) 계산기
- 컴퓨터는 0과 1을 조합(계산)하여 데이터 생성, 변경 등을 수행함.

## **컴퓨터 구조**
- 컴퓨터를 구성하는 동적 및 정적인 역할을 하는 부품들의 총칭
  - 동적인 부품(동작 담당) : CPU, 입출력장치
  - 정적인 부품(보관 담당) : 메모리, 보조기억장치
- 프로그램이 동작하는 공간이기 때문에 중요

## **CPU**
- 이진수의 연산을 수행하는 작업장
- CPU의 구성
  - ALU(산술연산장치) : 이진수 연산을 수행하는 장치, 실질적인 컴퓨터의 두뇌
  - 제어장치 : 레지스터의 연산자와 피연산자를 확인해 ALU에게 연산을 지시하는 장치
  - 레지스터 : 연산에 사용될 이진수로 구성된 연산자 및 피연산자가 저장되는 공간

## **주기억장치(Memory)**
- 현재 실행중인 프로그램의 명령어와 데이터가 저장되는 공간

## **보조기억장치**
- 실행중이지 않은 프로그램이 저장되는 공간

## **입출력장치**
- 컴퓨터와 인간간의 의사소통을 위한 장치
- 제어 신호를 발생시켜 제어 장치에게 보내 특정 동작을 수행함.

## **메인보드**
- 컴퓨터 부품들이 정보를 주고받을 수 있도록 통로를 제공하는 장치

## **시스템 버스**
- 컴퓨터 부품들이 정보를 주고받는 통로

## **비트(Bit)**
- 컴퓨터가 이해할 수 있는 최소 정보 단위
- 컴퓨터와 인간 간의 실질적인 대화 수단

## **워드(Word)**
- CPU가 한 번에 처리할 수 있는 정보량('이해할 수 있는'이 아닌 '처리할 수 있는')
- 보통 32비트나 64비트

## **인코딩(Incoding)과 디코딩(Decoding)**
- 인코딩 : 인간이 가진 정보를 컴퓨터가 이해할 수 있도록 비트로 변환하는 과정
- 디코딩 : 컴퓨터가 가진, 비트로 이루어진 정보를 인간이 이해할 수 있도록 변환하는 과정

## **아스키(ASCII)**
- 인간과 컴퓨터 사이의 문자 번역을 위한 번역기 혹은 사전 같은 존재, 서로에게 전달할 정보를 아스키를 통해 번역 후 전달함.
- 번역기가 번역할 수 없는 문자는 전달 할 수 없음.
- 아스키 속 문자(아스키 코드)는 각각 7비트로 표현됨

## **저급 언어와 고급 언어**
- 저급 언어 : 컴퓨터가 이해할 수 있는 언어(명령어)
  - 기계어 : 비트로 이루어진, 컴퓨터만을 위한 언어
  - 어셈블리어 : 기계어를 인간이 읽기 편한 형태로 번역한 언어
- 고급 언어 : 저급 언어를 인간이 이해하기 쉽게 만든 언어(소스 코드)
  - 컴파일 언어 : 소스 코드 전체가 저급 언어로 변환되는 실행되는 언어
  - 인터프리터 언어 : 소스 코드 한 줄씩 저급 언어로 변환되고 실행되는 언어
- 인간은 소스 코드를 작성하고, 작성된 소스 코드는 명령어로 변환되어 컴퓨터에게 전달되고 실행됨.

## **목적 코드(Object code)**
- 소스 코드를 컴파일하여 만들어진 저급 언어를 일컫는 말

## **주소 지정 방식**
- 명령어에 사용될 데이터를 데이터 값이 아닌 데이터가 저장된 주소로 표현하는 방식
- 명령어의 크기가 제한적이라서 데이터 자체를 명령어에 담기 보다는 크기가 작은 데이터가 저장된 주소를 담음.

## **클럭(Clock)**
- 컴퓨터가 작업하는 박자
- 클럭에 맞춰서 컴퓨터의 부품들이 작업을 진행함.

## **명령어 사이클**
- 명령어를 처리하는 단계
- 인출 -> 실행에 간접(메모리 접근), ISR(인터럽트 서비스 루틴)이 추가되는 구조

## **인터럽트(Inturpt)**
- CPU의 정상적인 실행 흐름을 방해하는 제어 신호
- 인터럽트의 종류 :
  - 동기 인터럽트(내부) : 예외라고도 하며, 컴퓨터 내부에서 발생시키는 인터럽트, CPU가 발생시키기 때문에 자동 동기화된다고 볼 수 있음.
  - 비동기 인터럽트(외부) : 컴퓨터 외부(입출력장치)에서 발생하는 인터럽트, 외부에서 발생하기 때문에 CPU와 동기작업이 필요함.

## **인터럽트 서비스 루틴(ISR)**
- 비동기 인터럽트를 처리하기 위한 프로그램으로써 메모리에 저장되어 있다.
- 입출력장치마다 ISR이 따로 존재한다.

## **인터럽트 벡터**
- 인터럽트의 식별 정보
- CPU는 인터럽트 벡터를 활용하여 ISR을 실행함

## **코어(Core)**
- ALU + 제어장치 + 레지스터 세트

## **소프트웨어적 스레드**
- 하나의 프로그램 내에서 독립적으로 실행될 수 있는 작업 흐름

## **하드웨어적 스레드**
- 소프트웨어적 스레드 내에서 코어가 한 번에 처리할 수 있는 명령어의 수
- 레지스터 세트의 수와 같음.

## **명령어 병렬 처리**
- CPU의 쉬는 시간을 줄여, 효율을 높이는 방법
- 파이프 라인, 슈퍼 스칼라, 비순차적 명령어 처리 등이 있음.

## **파이프 라인**
- 명령어 처리 과정을 인출, 실행 등으로 나누고, 같은 과정이 겹치지 않는 선에서 명령어를 겹처서 실행하는 기법

## **슈퍼 스칼라**
- 여러 파이프 라인을 두고 명령어를 처리하는 기법

## **비순차적 명령어 처리**
- 명령어를 비순차적으로 처리하여 파이프 라인의 정지를 방지하는 기법

## **명령어 집합(Instruction Set Architecture)**
- CPU가 이해할 수 있는 명령어 모음, CPU의 언어이기 때문에 CPU마다 다를 수 있음.
- 하드웨어가 소프트웨어를 이해하는 방식

## **CISC(Complex Instruction Set Computer)**
- 가변 길이 명령어를 사용하는 CPU

## **RISC(Reduced Instruction Set Computer)**
- 고정 길이 명령어를 사용하는 CPU

## **물리 주소**
- 하드웨어에 부여된 절대 주소

## **논리 주소**
- 소프트웨어에 부여된 상대 주소

## **저장 장치 계층 구조(Memory Hierarchy)**
- CPU와의 거리를 기준으로 저장 장치를 계층화하여 표현한 구조

## **캐시(Cache)**
- CPU의 연산 속도와 메모리 접근 속도의 차이를 줄이기 위한 저장 장치

## **캐시 히트(Cache Hit)**
- 캐시 속 데이터가 CPU에서 할용되는 경우

## **참조 지역성의 원리**
- 캐시가 메모리로부터 가져올 데이터를 결정하는 원리
- 다음 두 가지 지역성을 활용해 결정함.
  - 공간 지역성 : CPU가 접근했던 메모리 근처로 접근하려는 경향
  - 시간 지역성 : CPU가 최근에 접근했던 메모리에 다시 접근하려는 경향

## **하드 디스크(Hard Disk Drive)**
- 자석의 성질을 이용해 데이터를 저장하는 보조기억장치
- 하드 디스크의 구성 :
  - 플래터 : 데이터가 저장되는 공간, 자기 물질(작은 자석)로 구성되고, 이 물질들이 N극과 S극, 즉 0과 1을 나타냄, 원형 모양
  - 스핀들 : 플래터를 회전시켜 고정된 헤더가 작업을 할 수 있게 하는 장치
  - 헤드 : 데이터를 읽고 쓰는 장치
  - 디스크 암 : 헤드를 이동시키는 장치
  - 트랙 : 플래터를 여러 동심원으로 나눈 것 중 하나
  - 섹터 : 트랙의 한 조각, HDD의 가장 작은 전송단위
  - 실린더 : 여러 겹의 플래터 상의 트랙을 연결한 단위, 연속된 정보 저장 시 디스크 암의 이동거리가 줄어듬.

## **플래시 메모리**
- 전기 신호로 데이터를 읽고 쓰는 보조기억장치
- 플래시 메모리의 구성 :
  - 셀 : 플래시 메모리에서 데이터를 저장하는 가장 작은 단위
  - 페이지 : 셀들의 묶음, 읽기/쓰기의 단위
  - 블록 : 페이지들의 묶음, 삭제의 단위
  - 플레인 : 블록들의 묶음
  - 다이 : 플레인들의 묶음

## **RAID(Redundant Array of Independent Disks)**
- 여러개의 물리적 보조기억장치를 하나의 논리적 보조기억장치로 사용하는 기술

## **장치 컨트롤러**
- CPU와 입출력 장치를 연결하는 하드웨어
- 장치 컨트롤러를 통해 CPU와 입출력 장치가 데이터를 교환함.

## **버퍼링(Buffering)**
- 전송률이 다른 장치 간의 데이터를 버퍼라는 공간을 활용하여, 전송률을 맞추는 기술

## **장치 드라이버**
- CPU와 입출력 장치를 연결하는 소프트웨어

## **입출력 방법**
- 프로그램 입출력 :
  - 프로그램 속 명령어로 입출력을 제어하는 방법
  - 메모리 맵 입출력(Memory-Mapped I/O) :
    - 컴퓨터의 메모리와 장치 컨트롤러의 주소 공간을 하나로 간주하는 방식
  - 고립형 입출력(Isolation I/O) :
    - 컴퓨터의 메모리와 장치 컨트롤러의 주소 공간을 따로 간주하는 방식
- 인터럽트 기반 입출력 :
  - 인터럽트(제어 신호)를 통해 입출력을 제어하는 방법
- DMA 입출력 :
  - CPU를 거치지 않고 입출력 장치가 메모리에 바로 접근하여 입출력을 제어하는 방법

## **운영체제**
- 하드웨어를 조작하는 프로그램이자 명령어의 집합
- CPU는 운영체제의 명령어를 읽어 제어신호를 보냄.
- 내 프로그램이 실행되는 공간인 하드웨어에 접근하는 프로그램이기 때문에 중요함.

## **커널 영역(Kernel space)**
- 운영체제가 적재되는 메모리 상의 공간

## **사용자 영역(User space)**
- 사용자가 실행할 프로그램이 적재되는 메모리 상의 공간
- 커널 영역을 제외한 메모리 공간

## **커널**
- 운영체제의 핵심서비스(하드웨어 조작, 접근 / 프로그램 실행)를 담당하는 부분(명령어)

## **이중 모드**
- CPU의 명령어 실행모드를 2가지로 나눈 것
  - 커널 모드 : 메모리의 커널 영역의 명령어(운영체제)를 실행하는 것
  - 사용자 모드 : 메모리의 사용자 영역의 명령어(응용 프로그램)를 실행하는 것

## **시스템 호출**
- 응용 프로그램이 운영체제의 명령어를 실행하기 위해 CPU를 호출하는 것

## **프로세스**
- 현재 실행중인, 메모리에 적재된 프로그램
- 사용자가 볼 수 있는 포그라운드 프로세스와 볼 수 없는 백그라운드 프로세스가 있음.

## **PCB(Process Controll Block)**
- 프로세스 관리를 위해 커널 영역에 생성되는 일종의 태그

## **문맥(Context)**
- 프로세스를 실행하기 위한 정보
- 문맥 교환(Context Switching) : 실행 중인 프로세스를 다른 프로세스로 전환하는 것. PCB에 문맥을 백업하고, 가져오는 방식으로 진행됨.

## **메모리 영역**
- 정적 할당 영역 : 할당된 메모리 공간의 크기가 변하지 않는 영역
  - 코드 영역 : 프로그램의 명령어가 저장되는 공간
  - 데이터 영역 : 프로그램이 실행되는 동안 유지되는 데이터를 저장하는 공간(전역 변수)
- 동적 할당 영역 : 할당된 메모리 공간의 크기가 변하는 영역
  - 힙 영역 : 사용자가 직접 할당 가능한 공간, 반환이 필수적임.
  - 스택 영역 : 일시적인 데이터를 저장하는 공간(매개 변수, 지역 변수)

## **CPU 스케줄링**
- 운영체제가 프로세스에게 CPU를 공정하게 할당하는 것

## **스케줄링 큐**
- 자원을 사용하기 위해 프로세스들이 대기하는 공간

## **선점형 스케줄링**
- 대기 순서에 상관없이 우선 순위가 높은 프로세스에게 자원을 할당하는 것

## **비선점형 스케줄링**
- 우선 순위에 상관없이 대기 순서대로 프로세스에게 자원을 할당하는 것

## **호위 효과(Convoy Effect)**
- CPU 사용 시간이 짧은 프로세스 앞에 CPU 사용 시간이 긴 프로세스가 와서 오랜 시간 기다리는 것

## **기아 현상(Starvation)**
- 우선 순위가 낮은 프로세스가 우선 순위가 높은 프로세스에게 계속 밀려 CPU을 사용하지 못하는 현상

## **에이징(Aging)**
- 시간이 지남에 따라 우선 순위가 낮은 프로세스의 우선 순위를 높혀 기아 현상을 해결하는 방법

## **동기화(Synchronization)**
- 작업 상태를 공유하여 수행 시기를 맞추는 것
- 동기화를 위한 3원칙
  - 상호 배제
  - 진행(Progress) : 임계 구역이 비었을 때, 프로세스는 들어갈 수 있음.
  - 유한 대기(Bounded waiting) : 임계 구역에 들어가려하는 프로세스는 무한 대기하지 않고, 언젠가는 임계 구역에 들어갈 수 있음.

## **상호 배제(Mutual exclusion)**
- 공유 불가능한 자원의 동시 사용을 막는 것

## **공유 자원(Shared resources)**
- 여러 프로세서가 함께 사용하는 자원(전역 변수, 파일, 입출력장치, 보조기억장치..)

## **임계 구역(Critical section)**
- 동시에 실행되면 안되는 자원에 접근하는 코드 영역

## **레이스 컨디션(Race condition)**
- 여러 프로세스가 동시에 임계 구역의 코드를 실행한 상황

## **뮤텍스 락(Mutex lock : Mutual exclusion lock)**
- 공유 불가능한 자원에 동시에 접근하지 않도록 하는 상호 배제의 도구

## **바쁜 대기(Busy waiting)**
- 임계 구역에 진입하기 위해 대기 중에 임계 구역이 진입해도 되는지 계속 확인하는 상태

## **세마포(Semaphore)**
- 공유 자원이 여러 개 있을 때의 동기화 도구

## **모니터(Monitor)**
- 조건 변수를 사용한 동기화 도구

## **교착 상태(Dead lock)**
- 여러개의 프로세스가 서로의 자원을 얻으려고 무한정 대기하는 상태
- 교착 상태 발생 조건
  - 상호 배제
  - 점유와 대기 : 자원을 할당 받은 상태에서 다른 자원을 할당 받기를 기다리는 상태
  - 비선점
  - 원형 대기 : 프로세스가 원의 형태로 자원을 얻으려 대기하는 상태

## **스와핑(Swapping)**
- 사용되지 않는 프로세스가 메모리에서 보조기억장치로 가고, 사용될 프로세스가 보조기억장치에서 메모리로 이동하는 메모리 관리 기법
  - 스왑 공간(Swap space) : 사용되지 않는 프로세스가 내려오게 되는 보조기억장치 속 공간
  - 스왑 아웃(Swap-out) : 프로세스가 메모리에서 보조기억장치로 이동하는 것
  - 스왑 인(Swap-in) : 프로세스가 보조기억장치에서 메모리로 이동하는 것

## **메모리 할당**
- 프로세스에 메모리 공간을 할당하는 것
  - 최초 적합(First fit) : 메모리 내의 빈 공간을 순서대로 탐색하다가 프로세스를 적재할 수 있는 공간을 발견하면 해당 공간에 프로세스를 적재하는 것
  - 최적 적합(Best fit) : 메모리 내 프로세스가 들어갈 수 있는 가장 작은 공간에 프로세스를 할당하는 것
  - 최악 적합(Worst fit) : 메모리 내 프로세스가 들어갈 수 있는 가장 큰 공간에 프로세스를 할당하는 것

## **외부 단편화(External fragmentation)**
- 프로세스에 메모리를 할당하는 과정에서 프로세스를 할당할 수 없는 작은 공간이 생겨 메모리가 낭비되는 현상

## **가상 메모리**
- 프로그램 일부만 메모리에 적재하여 실제 물리 메모리보다 큰 프로세스를 실행할 수 있게 하는 기술

## **페이징(Paging)**
- 프로세스의 논리 주소 공간을 페이지라는 일정한 단위로 나눈 뒤, 메모라의 물리 주소 공간인 프레임이라는 페이지와 동일한 크기의 단위에 할당하는 가상 메모리 관리 기법
  - 페이지 아웃(Page-out) : 사용되지 않는 페이지를 메모리에서 보조기억장치로 이동시키는 것
  - 페이지 인(Page-in) : 사용할 페이지를 보조기억장치에서 메모리로 이동시키는 것

## **페이지 테이블**
- 페이지가 존재하는 메모리 상의 물리주소를 적어놓은 표
- CPU는 페이지 테이블을 활용해 연속된 것처럼 프로세스를 실행할 수 있음.

## **내부 단편화(Internal fragmentation)**
- 프로세스를 페이지 단위로 나눌 때, 페이지를 다 채우지 못한 채 프레임에 적재되어 메모리가 낭비되는 현상

## **요구 페이징(Demand paging)**
- 실행에 필요한(요구되는) 페이지만 메모리에 적재하는 것

## **스래싱(Thrashing)**
- 페이지 교체가 자주 일어나 CPU의 효율이 떨어지는 현상
