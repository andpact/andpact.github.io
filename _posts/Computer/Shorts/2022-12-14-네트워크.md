## **네트워크**
- 두 개 이상의 객체를 연결한 구조, 컴퓨터를 연결하면 컴퓨터 네트워크가 된다.

## **인터넷**
- 전 세계의 크고 작은 컴퓨터 네트워크를 모두 연결한 네트워크

## **패킷(Packet)**
- 컴퓨터 네트워크에서 사용하는 가장 작은 데이터 단위

## **대역폭(Bandwitdh)**
- 단위 시간당 최대 속도로 전송할 수 있는 정보량

## **디지털 데이터**
- 컴퓨터가 이해할 수 있는 0과 1의 집합

## **비트(Bit)**
- 디지털 데이터의 최소 단위

## **바이트(Byte)**
- 비트를 8개 모아서 표현한 디지털 데이터

## **근거리 통신망(Local Area Network)**
- 지리적으로 제한된 범위 내의 컴퓨터를 연결한 네트워크

## **광역 통신망(Wide Area Network)**
- 근거리 통신망보다 넓은 범위에 구축된 네트워크

## **인터넷 서비스 제공자(Internet Service Provider)**
- 광역 통신망을 구축할 수 있도록 네트워크를 제공하는 자

## **인터넷 공유기(Broadband Router)**
- 근거리 통신망에서 광역 통신망으로 접속하게 해주는 가정용 라우터

## **DMZ(DeMilitarized Zone)**
- 외부에 공개하기 위한 네트워크로 외부에서 접속이 가능한 네트워크이다.

## **온프레미스(On-premise)**
- 사내 또는 데이터 센터에 서버를 두고 운영하는 것

## **프로토콜(Protocol)**
- 네트워크 통신에서 필요한 작업(OSI 모델)의 내부에서 사용하는 규칙

## **OSI 모델(Open Standards Interconnection Model)**
- 네트워크로 통신할 때 필요한 작업 여거가지를 순서를 부여한 모델
- OSI 모델 7계층 :
  - 응용 계층
  - 표현 계층
  - 세션 계층
  - 전송 계층
  - 네트워크 계층
  - 데이터 링크 계층
  - 물리 계층

## **TCP/IP 모델**
- OSI 모델의 7계층을 4계층으로 표현한 모델
- TCP/IP 모델 4계층 :
  - 응용 계층
  - 전송 계층
  - 인터넷 계층
  - 네트워크 접속 계층

## **캡슐화(Encapsulation)**
- 네트워크 통신 시 송신 측에서 OSI 모델의 상위 계층에서 하위 계층으로 내려갈 때 헤더를 붙이는 과정

## **역캡슐화(Decapsulation)**
- 네트워크 통신 시 수신 측에서 OSI 모델의 하위 계층에서 상위 계층으로 올라갈 때 헤더를 제거하는 과정

## **헤더(Header)**
- OSI 모델 각 계층에서의 작업 내용을 담은, 데이터의 앞부분에 추가되는 정보

## **트레일러(Trailer)**
- 네트워크 통신에서 데이터를 전송하기 전에 데이터의 마지막에 추가되는 정보

## **랜 카드(Network Interface Controller)**
- 네트워크를 통해서 통신할 수 있도록 신호를 주고받는 하드웨어

## **물리 계층(Physical Layer)**
- 컴퓨터와 네트워크 장비를 연결하고, 디지털 데이터를 전기 신호로 변환하는 계층

## **트위스티드 페어 케이블(Twisted Pair Cable)**
- 구리 선 여덟 개를 두 개씩 꼬아서 만든, 네트워크 통신에 사용되는 유선 전송 매체
  - UTP 케이블(Unshielded TPC)
  - STP 케이블(Shielded TPC)
- 랜 케이블(LAN Cable, 랜 선)로도 불린다.
  - 다이렉트 케이블 : 구리 선 여덟 개를 같은 순서로 연결한 케이블, 한 방향 통신 가능
  - 크로스 케이블 : 한 쪽 커넥터의 1, 2번 구리선을 다른 쪽 커넥터의 3, 6번에 연결한 케이블, 양방향 통신 가능

## **리피터(Repeater)**
- 전기 신호를 복원하고, 증폭시켜주는 물리 계층의 장비

## **허브(Hub)**
- 컴퓨터 여러 대를 서로 연결하고 또한 리피터의 역할을 수행하는 물리 계층의 장비

## **데이터 링크 계층(Data Link Layer)**
- LAN에서 데이터를 정상적으로 주고받기 위한 계층

## **이더넷(Ethernet)**
- 데이터 링크 계층에서 사용하는 프로토콜

## **MAC 주소(Media Access Controll Address)**
- 랜카드를 구별하기 위한 중복되지 않는 물리 주소

## **프레임(Frame)**
- 이더넷 헤더와 트레일러가 추가된 데이터

## **스위치(Switch)**
- 허브의 데이터 충돌을 방지하는 데이터 링크 계층의 장비

## **MAC 주소 테이블**
- 스위치의 포트 번호와 해당 포트에 연결된 컴퓨터의 MAC주소가 등록되는 데이터베이스

## **플러딩(Flooding)**
- 여러 컴퓨터가 연결된 네트워크에서 원치 않는 컴퓨터로 데이터가 전송되는 현상

## **전이중 통신 방식**
- 데이터 송수신을 동시에 처리하는 방식

## **반이중 통신 방식**
- 회선 하나로 송수신을 번갈아 처리하는 방식

## **충돌 도메인(Collision Domain)**
- 데이터 전송 시 충돌이 발생했을 때, 그 영향이 미치는 범위

## **네트워크 계층(Network Layer)**
- 네트워크 간의 통신을 하기 위한 계층

## **라우터(Router)**
- 네트워크(LAN) 간의 통신에서 경로를 찾아주는 네트워크 계층의 장비

## **IP(Internet Protocol)**
- 네트워크 계층에서 사용하는 프로토콜

## **IP 패킷(IP Packet)**
- IP 헤더가 추가된 데이터

## **IP 주소**
- IPv4 : 32비트로 이루어진 IP 주소
- IPv6 : 128비트로 이루어진 IP 주소
- 네트워크 ID : IP 주소 내에서 네트워크를 나타내는 부분
- 호스트 ID : IP 주소 내에서 실제 컴퓨터를 나타내는 부분

## **IP 주소 클래스**
- A 클래스 : 네트워크 ID의 크기 보다 호스트 ID의 크기 가 큰 IP 주소, 대규모 네트워크 주소
- B 클래스 : 네트워크 ID의 크기와 호스트 ID의 크기가 동일한 IP 주소, 중형 네트워크 주소
- C 클래스 : 네트워크 ID의 크기보다 호스트 ID의 크기가 작은 IP 주소, 소규모 네트워크 주소
- D 클래스 : 멀티캐스트(Multicast) 주소
- E 클래스 : 연구 및 특수용도 주소

## **네트워크 주소**
- 네트워크 전체를 대표하는 IP 주소

## **브로드캐스트 주소**
- 네트워크 내의 모든 컴퓨터에 데이터를 전송하기 위한 IP 주소

## **서브넷(Subnet)**
- 큰 네트워크에서 분할된 작은 네트워크

## **서브넷 마스크(Subnet Mask)**
- 서브넷을 만들 때, 네트워크 ID만 나타내어 같은 네트워크인지 판별하게 하는 마스크

## **게이트웨이(Gateway)**
- 다른 네트워크와 통신하기 위한 라우터의 주소

## **라우팅(Routing)**
- 현재 네트워크에서 다른 네트워크에 접속하기 위한 최적의 경로를 찾는 과정

## **멀티캐스트(Multicast)**
- 한 컴퓨터에서 여러 컴퓨터로 동시에 패킷을 전송하는 방식

## **브로드캐스트(Broadcast)**
- IP 네트워크에 있는 모든 컴퓨터로 패킷을 전송하는 방식

## **전송 계층(Transport Layer)**
- 목적지에 신뢰할 수 있는 데이터를 전달하기 위한 계층

## **연결형(Connection-oriented) 통신**
- 상대편을 확인하면서 통신하여 신뢰할 수 있고 정확한 데이터를 전달하는 통신

## **비 연결형(Connetionless) 통신**
- 상대편을 확인하지 않고 일방적으로 데이터를 전달하는 통신

## **TCP(Transmission Control Protocol)**
- 연결형 통신에서 사용하는 프로토콜

## **세그먼트(Segment)**
- 전송게층에서 TCP 헤더가 붙은 데이터

## **3-way 핸드셰이크**
- 데이터를 보내기 전에 연결을 확립하기 위해 패킷을 세 번 교환하는 것

## **포트 번호(Port Number)**
- 프로그램을 식별하기위해 프로그램에 할당된 번호

## **UDP(User Datagram Protocol)**
- 비 연결형 통신에서 사용하는 프로토콜

## **UDP 데이터그램**
- 전송계층에서 UDP 헤더가 붙은 데이터

## **응용계층(Application Layer)**
- 어플리케이션과 데이터를 주고받기 위한 계층

## **HTTP(HyperText Transfer Protocol)**
- 웹사이트를 불러오기 위해 사용하는 응용계층 프로토콜

## **FTP(File Transfer Protocol)**
- 파일을 전송하기 위해 사용하는 응용계층 프로토콜

## **SMTP(Simple Mail Transfer Protocol)**
- 메일을 송신하기 위해 사용하는 응용게층 프로토콜

## **POP3(Post Office Protocol)**
- 메일을 수신하기 위해 사용하는 응용계층 프로토콜

## **DNS(Domain Name System)**
- 컴퓨터에 붙여진 이름(도메인)을 기반으로 IP 주소를 알아내는 응용계층 프로토콜

## **URL(Uniform Resource Locator)**
- 도메인으로 서버의 데이터를 불러오기 위한 주소

## **무선 액세스 포인트(Wireless Access Point)**
- 무선 인터넷 접속을 도와주는 중계 장치

## **SSID(Service Set IDentifier)**
- 무선 랜을 다른 무선 랜과 구분하기 위해 패킷의 헤더에 붙는 식별자

## **채널(Chnnel)**
- 무선 통신에서 사용되는 분리된 전송로
